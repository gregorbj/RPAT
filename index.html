Overview

State and metropolitan area governments are facing substantial new challenges when planning for the future. They can no longer assume that past trends will continue into the future given the emergence of new transportation technologies, such as self-driving cars, and pressing problems, such as the need to greatly limit carbon emissions. Moreover, there are many questions about how the future may unfold such as: how much will the price of gas increase? will millennials continue to drive less than previous generations as they age? and will the availability of self-driving cars increase driving?

These challenges and uncertainties require a strategic approach to planning in which many possibilites for future trends are considered along with many potential policies and actions for influencing those trends and their effects on communities. Analytical methods and tools need to be able to consider many factors of interest and need to be executed rapidly so that many potential futures and courses of action may be analyzed. In addition, planners need to have the means to efficiently sift through and make sense of the results of those analyses.

This document provides an overview of a pilot project to create methods for developing and analyzing hundreds of future transportation and land use scenarios using the Rapid Policy Assessment Tool (RPAT). It also demonstrates an interactive visualization tool that planners and others may use to explore and make sense of the results. All project documents and code are available in the project's GitHub repository: here.

Rapid Policy Assessment Tool (RPAT)

The Rapid Policy Assessment Tool is a strategic planning model for metropolitan areas that enables users to analyze the effects of different land use and transportation scenarios on a number of outcomes such as emissions, congestion, vehicle travel, and costs. The RPAT was developed as part of the SHRP 2 C16 project, which provided tools, methods, and resources for transportation planning agencies in the United States to evaluate the effects of smart growth policies on travel demand. The following links provide additional information about the RPAT.

  • SHRP2 C16 report on the effect of smart growth policies on travel demand

Running Multiple RPAT Scenarios

Although the RPAT software runs quickly, the interface is not designed to enable users to easily set up and run large numbers of scenarios. The pilot project developed methods and modified the RPAT software to overcome this limitation. A test was executed in which over 300 scenarios were developed and analyzed using the RPAT. In addition, a data visualization application was developed to display the results of the analysis. A summary of the methods developed for creating and running multiple scenarios is presented below. The data visualization application is presented in the next tab of this web page.

The RPAT software is operated with a graphical user interface (GUI) that is designed to assist users with organizing and formatting inputs, starting the model, and viewing summary results. Although the GUI makes it easier for RPAT users to operate the tool without errors, it forces users to set up and run one scenario at a time. This is a cumbersome way to set up and evaluate a large number of scenarios. Fortunately, the GUI is a layer on top of several R language scripts that perform all of the model calculations. Moreover, all scenario inputs are stored in text files that are easily edited in a spreadsheet program. This enables a scenario development and model run process to be scripted and run using the R command interface. Two R scripts were created to do this: make_scenarios.r, and run_many_scenarios.r. These scripts and documentation are available in the project's GitHub repository.

The RPAT installation program installs a number of R and Python scripts. Only 3 of the RPAT R scripts are needed to carry out the method summarized here and documented in the GitHub repository. In addition, The demo input files are needed to provide templates for the scenario inputs. The file structure needed to organize these files, the RPAT scripts, and the make_scenarios.r and run_many_scenarios.r scripts are described in the project documentation.

The make_scenarios.r script uses a full factorial design approach for the development of scenarios. A user organizes inputs whose effects are to be studied in categories and then develops several sets of input files in each category to reflect different levels of influence. For example the demo used the following categories and levels:

  • B - Bikes/Light-weight Vehicles
    1. Base bike diversion (9.75%)
    2. Double bike diversion (19.5%)
  • C - Cost (vehicle mileage charge)
    1. Base (no charge)
    2. 4 cents per mile
    3. 8 cents per mile
  • D - Demand Management
    1. Base
    2. Double all participation rates
    3. Double all participation rates and transit subsidy level
  • L - Land Use
    1. Base (growth proportions by place type same as base year proportions)
    2. Halve suburban population growth and shift that portion to urban core
  • P - Parking
    1. Base (existing costs and existing proportions of trips affected)
    2. Increase trips affected by 20%
    3. Double parking cost as well as increase trips affected by 20%
  • T - Transit
    1. Base (supply stays at present level)
    2. Double transit supply
    3. Triple transit supply

The make_scenarios.r script creates a scenario for each combination of category levels along with input files that do not vary by scenario. The total number of scenarios created is the product of the number of levels in each category; 324 for the demo.

The run_scenarios.r script runs the RPAT model for each of the scenarios successively. After all of the runs have been completed, the script compiles the outputs produced by RPAT into a set of files, one for each of the performance measures calculated by RPAT, where results for all scenarios are included in each file. The script also produces a summary file of key performance measures for all scenarios that is used in the interactive data visualization.

About Scenario Sensitivity Testing

The sensitivity tests displayed were conducted to test the potential for modifying the RPAT software to run multiple scenarios and to display the results by modifying the scenario viewer web app developed for the Regional Strategic Planning Model (RSPM). Preliminary inputs for the Corvallis Metropolitan Planning Area (CAMPO) RPAT project were modified to create these scenarios. Selected inputs were changed to different degrees. Then all combinations of the input variations were run as scenarios. This web app shows the results for several summary outputs. Note that the scenarios were developed solely for the purpose of testing out software modifications for running multiple scenarios and for displaying those scenarios in this viewer. Read the "Quick Start" and "Detailed Instructions" for more information.

About the Scenario Viewer

This web page enables you to explore the results of these sensitivity tests. With it, you can select different combinations of factors and find out how various future outcomes are affected by your selection. You can also select desired outcomes and find out what factors cause those outcomes to occur. By experimenting with the selections and observing the results, you will get a better understanding of the relationships between policy choices and context factors and future outcomes.

Detailed Instructions

Categories

Many factors such as parking pricing and transit supply affect light duty vehicle travel. Because the number of factors considered is large, they were grouped into 6 categories as follows:

  • Bicycles The diversion of single-occupant vehicle travel to bicycles, electric bicycles and other light-weight vehicles.
  • Demand Management Programs and incentives which encourage people to drive less including ridesharing, van pooling, telecommuting, and transit subsidies.
  • Land Use The form in which development occurs (density, regional assessibility, mixed use, etc.) represented by the distribution of population and employment by place type.
  • Parking The extent of paid parking and its price.
  • Transit The extent and frequency of transit service.
  • Vehicle Travel Cost The combination of fuel prices and vehicle travel charges to pay for roadways and to pay for externalities such as carbon pricing.

Levels

Several levels were defined for each of the categories. These levels are numbered to indicate the amount of change from a reference case which represents the continuation of adopted local plans, policies and trends.

  • Level 1: Corresponds to current conditions.
  • Levels 2-3: Representing more ambitious policies.

The levels are displayed in pie charts, one for each category, as shown in the following illustration.

Each pie chart has a legend showing the color associated with each level. The sizes of the pie slices show the proportions of the selected scenarios in each each level. The number of selected scenarios in each level is shown in the corresponding pie slices. You can select (or deselect) the scenarios in a level by either clicking on the pie slice or the corresponding legend entry. The selected level is colored and the non-selected levels are grayed-out.

Outcomes

Given the chosen category inputs, the web page also shows future year outcomes for the following performance measures:

  • Fatalities & Injuries: annual traffic fatalities and injuries per 1000 persons.
  • Vehicle Cost Per Capita: average annual cost for owning & operating vehicles per person.
  • DVMT Per Capita: average daily vehicle miles traveled per person.
  • GHG Emissions Per Capita: average annual metric tons of greenhouse gas emissions per person.
  • Fuel Consumption: average annual gallons of gasoline and other fuels consumed per person.
  • DVHT Per Capita: average daily vehicle hours of travel per person.

Each outcome is illustrated in a bar chart showing the range of outcome values for the selected scenarios. The bar height indicates the number of selected scenarios with that outcome value. The average value for all of the selected scenarios is shown above the bar chart. The following illustrations shows what the Fatalities & Injuries, Vehicle Cost Per Capita, and DVMT Per Capita bar charts look like when all scenarios are selected.

The bar charts can also be used to make a selection of scenarios. This can be done by hovering the mouse cursor over the bar at one end of the desired selection range. When the cursor is in the shape of a crosshairs, click and drag the mouse cursor to the other end of the desired range. The bar chart will change to show the selected range. In addition, handles will appear at the ends of the selected range. You can click one these and drag it to alter the selection. You can also click on the middle of the selection (when the shape of the cursor is a crossed arrow) to drag the whole selection to a different location on the bar chart. As you make a selection in one bar chart, all the other bar charts will change to show just the selected scenarios. The pie charts will also change accordingly. The following illustration shows a selection made on the Fatalities & Injuries bar chart and the corresponding values for the other bar charts in the row. You can see in the picture the crossed-arrow cursor shape which means that the whole selection can be dragged to a new position.

Selections can be made simultaneously on multiple pie charts and bar charts. As more selections are made, all of the charts will be updated to show the selections that meet all of the selection conditions. To clear all of the selections and show all of the scenarios, just click on one of the Clear All Selections links on the web page.

Selected Scenario Table

Finally, at the bottom of the web page is a table which shows the data for the selected scenarios. The values in this table can be copied and saved in a spreadsheet. To select the data to copy, double-click in the top left-hand cell, press the Shift key, and then click in the bottom right-hand cell. Once the data has been selected, copy it by pressing the Ctrl c keys for a Windows computer or the Command c keys for an Apple Macintosh computer. Paste as unicode text into the spreadsheet of your choice.
